Movie API

Instructions for using Movie API

Request Method URL Response HTTP Method The format of the response data
Create a new User in the list /users/ POST no need of Authorization JSON format of user to be added in the list { "Username": "username string", "Password": "password example", "Email": "example@mail.com", "Birthday": "1990/01/01" (date format), } JSON format with the new user created JSON response from the list of users, able to create a new user in JSON file
Login in user to Authorization /users/login?Username=[username]&Password=[password] Insert data through PARAMS POST no need of Authorization no JSON format JSON format with a JWT TOKEN created (very long). JSON response from the list of users, able to create a JWT TOKEN
Create movie in the list /movies/ POST JSON format of movie or element to add in the list. { "Genre": { "Name": "Name of the Movie", "Description": "description of the movie" }, "Director": { "Name": "name director", "Bio": "information Bio from director" }, "_id": "60a8cd5037f59be3e29b1c9e", Normally generated when movie is created ramdonly "Title": "movie name", "Description": "description from the movie", "ImagePath": "link to the image" "featured": "if its featured in cinemas or not" }, JSON format added to the list Create new movie in JSON file.
request all the movie list /movies need Authorization from the user to call it. GET none: just need no call, not format needed JSON format of the complete list of movies JSON response returning the complete list of selected movies, no filter
Return all users /users GET need Authorization from the user to call it. TOKEN no format JSON format of selected movie JSON response selecting the movie through the director of the movie
Return user by name /users/:Username GET need Authorization from the user to call it. TOKEN no format JSON format of selected movie JSON response selecting the movie through the director of the movie
Request movie selected by title /movies/:Title GET need Authorization from the user to call it. TOKEN not format needed to call JSON format of selected movie JSON response returning a selected movies form the list of movies
Request movie selected by genre /genre/:Name GET need Authorization from the user to call it. JSON format of selected movie JSON response returning a selected title form the list of movies
Return movie selected by director name /director/:Name GET need Authorization from the user to call it JSON format of selected movie JSON response selecting the movie through the director of the movie
Update users selected by name /users/:Username PUT need Authorization from the user to call it. TOKEN write new strings of the movie in the JSON format { "Username": "new", "Password": "new", "Email": "new", "Birthday": "1990/01/01" (date format), "FavMovies" : [] } it will overwrite the previous description and return the new data as JSON format of selected movie JSON response returning a selected user updated
Add a movie to a user's list of favorites /users/:Username/Movies/:MovieID POST need Authorization from the user to call it. TOKEN no JSON format needed, its Updated through the URL it will add the new id directly on the Favorites movies JSON response returning a user updated
Delete user from the list of users DELETE/ /users/:Username no JSON format needed Succesful message that was deleted remove user in JSON file and from the list.
